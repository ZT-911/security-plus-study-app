THEORY SCREEN 1

Data Classification

Data classification determines how sensitive information is and what level of protection and handling it requires.

Core Concept
Data classification defines how sensitive information is and how it must be protected and handled. Organizations classify data to determine the appropriate level of security controls, access restrictions, storage requirements, and transmission rules.
Key Principle
The more sensitive the data, the stronger the required controls and handling rules.
Common Classification Levels

Public

No impact if disclosed.

Private / Internal

Intended for internal use only.

Confidential

Sensitive data that requires protection.

Restricted / Highly Confidential

Critical data with severe impact if exposed.

Exam Insight
Security+ often tests whether you can match a control, access level, or handling requirement to the correct classification level.
Why It Matters
Classification is the foundation for applying protection consistently. Without it, data handling becomes weak, expensive, or both.
THEORY SCREEN 2

Data States

Data exists in different operational states, and each state requires its own protection strategy.

Core Concept
Data exists in three main states: at rest, in transit, and in use. Identifying the state first often reveals which control is most appropriate.

Data at Rest

Stored data such as disks, databases, backups, and archived files.

Data in Transit

Data moving across networks between systems, services, or users.

Data in Use

Data actively being processed in memory by the CPU or running applications.

Security Controls by State

At Rest

Disk encryption, file permissions, access control.

In Transit

TLS, HTTPS, VPN.

In Use

Process isolation, session control, memory protection.

Key Insight
Data in Use is the hardest to protect because it must remain accessible in memory while it is being processed.
Exam Trap
Confusing Data in Transit with Data in Use is extremely common. Network movement is transit. Active RAM processing is use.
THEORY SCREEN 3

DLP + Encryption

These controls both protect data, but they solve different problems and should not be confused.

Data Loss Prevention (DLP)
DLP systems are designed to detect and prevent data exfiltration. They monitor sensitive data and enforce policies related to movement and transfer.

Common Actions

Blocking unauthorized transfers, preventing sensitive emails, monitoring user activity involving sensitive data.

Important Concept

DLP focuses on controlling and monitoring the movement of sensitive data.

Encryption Basics
Encryption protects the confidentiality of data by making it unreadable without the correct key.

Symmetric Encryption

One shared key, fast, example: AES.

Asymmetric Encryption

Public and private key pair, slower, example: RSA.

Key Difference
DLP helps detect and block the movement of sensitive data. Encryption protects the content from unauthorized reading.
Exam Trap
Encryption protects confidentiality, not availability or full access control. It also does not automatically stop data exfiltration.
THEORY SCREEN 4

Policies, Destruction & Risks

Data protection includes lifecycle policies, proper disposal, and understanding which controls reduce which risks.

Data Policies
Policies define how data is managed throughout its lifecycle, including retention, access, handling, and transmission.
Policy Focus Areas

Data Retention

How long data must be stored.

Access Control

Who is allowed to access the data.

Data Handling

How data can be used, shared, and transmitted.

Data Destruction Methods

Clearing

Basic logical deletion. Data may still be recoverable.

Purging

Secure deletion. Data is not recoverable through standard means.

Destruction

Physical damage such as shredding or incineration.

Common Risks
Data breaches
Unauthorized access
Misclassification
Human error
Exam Focus
Deleting a file does not mean the data is gone. Security+ often tests whether you can identify which control mitigates which lifecycle risk.
