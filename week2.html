THEORY SCREEN 1
Data States
Data exists in different operational states, and understanding those states is essential for choosing the right protection control.
Core Concept
Data exists in three main states: at rest, in transit, and in use. These states are connected, but each one introduces different exposure and therefore different security controls.
STATE 01
AT REST
Data stored on disks, databases, file systems, or backups.
disk encryption · storage controls
STATE 02
IN TRANSIT
Data moving across networks between systems, users, or services.
TLS · HTTPS · VPN
STATE 03
IN USE
Data actively being processed in memory by the CPU or running applications.
hardest to protect · memory exposure
Key Insight
Data in Use is the hardest state to protect because it must remain accessible while the system processes it.
Exam Trap
Confusing In Transit with In Use is common. Network movement is transit. Active processing in RAM is use.
THEORY SCREEN 2
Data Classification
Classification determines sensitivity level and drives the strength of protection, handling rules, and access restrictions.
Core Concept
Data classification defines how sensitive information is and what level of protection it requires. The more sensitive the data, the stronger the expected controls.
LEVEL 01

Public

Information with little or no impact if disclosed.

LEVEL 02

Private / Internal

Information intended for internal organizational use only.

LEVEL 03

Confidential

Sensitive data that requires strong protection and controlled access.

LEVEL 04

Restricted / Highly Confidential

Critical or regulated data with severe impact if exposed or mishandled.

Key Principle
Classification is what determines how data should be protected, handled, stored, and shared.
Exam Trap
Classification is based on sensitivity and impact, not simply on where the data is stored.
THEORY SCREEN 3
Data Roles
Security+ often tests who decides policy, who implements controls, and who simply uses the data.
Core Concept
Data protection is not only about technology. It also depends on role responsibility. The exam often expects you to distinguish decision-making from implementation.

OWNER

Defines classification, policy, acceptable use, and required protection level.

↓

CUSTODIAN

Implements storage, backups, permissions, and technical or administrative controls.

↓

USER

Accesses and uses the data according to the rules already defined and enforced.

Related Legal Roles
In some regulatory contexts, terms like controller and processor may also appear, especially when discussing legal responsibility for personal data handling.
Exam Trap
Owner decides. Custodian implements. Mixing those two is a classic wrong answer.
THEORY SCREEN 4
Data Protection System
Protection must work across the lifecycle, not as a single isolated control.
Core Concept
Data protection is a system. It involves storage, access, and sharing controls across the full lifecycle of the information.

STORE

Retention rules define how long data must remain stored and protected.

ACCESS

Access control determines who can view, modify, or manage information.

SHARE

Handling and transmission policies determine how data can move safely.

Destruction Methods

Clearing

Basic logical deletion. Data may still be recoverable.

Purging

Secure deletion. Data is not recoverable by standard means.

Destruction

Physical elimination such as shredding or incineration.

Common Risks
Data breaches, unauthorized access, misclassification, and human error remain the most common failure points.
Exam Trap
Deleting a file does not mean the data is gone. Clearing is not the same as secure destruction.
