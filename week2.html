THEORY SCREEN 1

Data Classification

Understand how data sensitivity determines protection level, handling rules, and access requirements.

Core Concept
Data classification defines how sensitive information is and how it must be protected and handled. Organizations classify data to determine the correct level of security controls, access restrictions, storage requirements, and transmission rules.
Key Principle
The more sensitive the data, the stronger the required controls and handling rules.
Common Classification Levels

Public

No impact if disclosed.

Private / Internal

Intended for internal use only.

Confidential

Sensitive data that requires protection.

Restricted / Highly Confidential

Critical data with severe impact if exposed.

Exam Insight
Security+ questions often test your ability to match a control, access level, or handling rule to the correct classification level.
Why It Matters
Classification is the foundation for consistent protection. Without it, organizations either overprotect data inefficiently or underprotect sensitive information.
THEORY SCREEN 2

Data States

Data exists in different states, and each state requires its own protection strategy.

Core Concept
Data exists in three main states: at rest, in transit, and in use. The correct control often depends on identifying the state first.

Data at Rest

Stored data such as disks, databases, and backups.

Data in Transit

Data moving across networks between systems or users.

Data in Use

Data actively being processed in memory by the CPU or applications.

Security Controls by State

At Rest

Disk encryption, file permissions, access control.

In Transit

TLS, HTTPS, VPN.

In Use

Process isolation, session control, memory protection.

Key Insight

Data in Use is the hardest to protect because it must remain accessible in memory while it is being processed.

Exam Trap
Confusing Data in Transit with Data in Use is extremely common. Network movement is transit. Active processing in RAM is use.
THEORY SCREEN 3

DLP + Encryption

These two controls are related to protecting data, but they solve different problems.

Data Loss Prevention (DLP)
DLP systems are designed to detect and prevent data exfiltration. They monitor sensitive data and enforce policies related to movement and transfer.
Common Actions
Blocking unauthorized transfers, preventing sensitive emails, monitoring user activity involving sensitive data.
Important Concept
DLP focuses on controlling and monitoring the movement of sensitive data.
Encryption Basics
Encryption protects the confidentiality of data by making it unreadable without the correct key.
Symmetric Encryption
One shared key, fast, example: AES.
Asymmetric Encryption
Public and private key pair, slower, example: RSA.
Key Difference
DLP helps detect and block the movement of sensitive data. Encryption protects the content itself from unauthorized reading.
Exam Trap
Encryption protects confidentiality, not availability or full access control. It also does not automatically stop data exfiltration.
THEORY SCREEN 4

Policies, Destruction & Risks

Data protection is not just storage. It includes lifecycle rules, secure destruction, and risk control.

Data Policies
Policies define how data is managed throughout its lifecycle, including retention, access, handling, and transmission.
Policy Focus Areas
Data Retention
How long data must be stored.
Access Control
Who is allowed to access the data.
Data Handling
How data can be used, shared, and transmitted.
Data Destruction Methods

Clearing

Basic logical deletion. Data may still be recoverable.

Purging

Secure deletion. Data is not recoverable through standard means.

Destruction

Physical damage such as shredding or incineration.

Common Risks
Data breaches
Unauthorized access
Misclassification
Human error
Exam Focus
Deleting a file does not mean the data is gone. Security+ often tests whether you can identify which control reduces which lifecycle risk.
